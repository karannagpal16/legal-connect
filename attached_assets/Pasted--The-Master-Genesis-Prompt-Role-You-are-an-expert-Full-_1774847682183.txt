
The Master Genesis Prompt

Role: You are an expert Full-Stack Developer and Legal-Tech Architect.

Project: Build 'RNA Legal Connect' for the law firm 'Rishika Nagpal & Associates' (Subhash Nagar, Delhi).

1. Core Visual Identity:

Theme: Sophisticated Professional. Use a palette of Deep Navy (#001f3f), Charcoal Grey (#333333), and Brushed Gold accents (#D4AF37).

Typography: Use a clean, authoritative Serif font for headings (like Playfair Display) and a highly readable Sans-Serif for data (like Inter).

Mobile First: The UI must be optimized for a lawyer walking through a crowded Delhi court corridor. Buttons must be large and thumb-friendly.

2. Database Architecture (PostgreSQL):

Create the following schema to ensure data sovereignty and DPDP Act compliance:

Users Table: id, name, bar_id, email, phone, role (Admin/Associate/Intern/Proxy), location_base.

Cases Table: id, title, case_number, court_name (Dropdown: Tis Hazari, Patiala House, Saket, Rohini, Dwarka, Karkardooma, Delhi High Court), court_room_no, judge_name, next_date, status (Pending/Disposed).

Proxy_Tasks Table: id, case_ref, location, task_type (Pass-over, Adjournment, Evidence, Arguments), fee_amount, assigned_to_id, status (Open, Accepted, Completed).

Intern_Quests Table: id, title, description, xp_points, deadline, status.

3. Functional Modules to Build Now:

Auth: Secure Login/Signup. Admins (Rishika) must be able to approve new users.

The Auto-Diary (Professional Half): A dashboard for the user to see their 'Cases for Today' and 'Upcoming Hearings'. Include an 'Add Case' form.

The Proxy Marketplace: A real-time feed where I can post a task from my diary and other verified 'Proxy' users can click 'Accept'.

The Revenue Tracker: A hidden 'Founder Analytics' page that calculates:

Total Managed Revenue (Active Cases * ₹50,000).

Marketplace Profit (10% of completed Proxy Tasks).

A progress bar toward a ₹3.8 Crore 'Singapore Expansion' goal.

4. Technical Constraints:

Security: Use AES-256 encryption for any document links.

Compliance: Include a 'Consent Manager' checkbox for the Indian DPDP Act 2023 on the signup page.

Offline First: Ensure the 'Daily Diary' is cached so it can be viewed in court basements with poor signal.

How to handle the AI after you paste this:

The AI will ask: "Should I start with the UI or the Database?"

Your Answer: "Start by setting up the Database and the Login page first. I want to be able to create an account immediately."

The AI will show you a link: Open it on your phone.

Your Action: If the gold color looks "yellow" or cheap, say: "Change the gold accent to a more muted, metallic brushed gold."

The AI will ask about APIs: * Your Answer: "For Day 1, use 'Mock Data' (fake data) for the court dates. We will connect the real e-Courts API once the interface is perfect."

Why this prompt works:

It tells the AI who you are (a founder in Subhash Nagar), which sets the professional tone.

It defines the math (the ROI tracking), so the app isn't just a list; it’s a financial tool.

It anticipates the Delhi Court names, saving you from typing them in later.
